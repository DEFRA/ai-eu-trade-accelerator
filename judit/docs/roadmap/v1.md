# Roadmap

Authoritative sequencing (v1 → Future) is in [`docs/canonical/roadmap.md`](../canonical/roadmap.md). This page summarises **v1 delivery** and pointers.

## v1 — In scope

- proposition-first legal analysis domain model
- source-first ingestion via source registry references
- single-jurisdiction proposition inventory workflow
- divergence workflow built on propositions as downstream mode
- FastAPI backend
- analysis workbench (`/`) and operations/registry inspector (`/ops`) split
- operations/audit endpoints for runs, sources, traces, and review decisions
- temporal history inspection for sources, propositions, and divergence findings (APIs / ops; not a dedicated “timeline” analysis screen)
- LiteLLM gateway
- export bundle
- docs site with generated architecture visuals
- D2 diagrams wired into docs/decks
- Marp deck with generated visual slides

## v1.5 status (summary)

Reviewable structured propositions: reliability, provision types, quieter scope links, broader review decisions, article/provision grouping. **No timeline view in product UI until that foundation is solid** — see canonical roadmap.

Status snapshot (2026-05, factual):

- structured proposition reliability: implemented, tested, operator-friendly gap
- provision type surfacing: implemented, tested, documented
- broader review decisions (proposition/structured/scope/completeness): implemented, tested, documented
- article/provision grouping: implemented, tested, documented
- timeline view in product UI: planned
